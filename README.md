# دليل استخدام Git & GitHub 🚀

دليل تعليمات شامل ومبسط لتعلم كيفية التعامل مع الغيت ورفع المشاريع.

---

## 1. البداية والتحميل

- **تحميل الأداة:** نقوم بتحميل **Git Bash** من الرابط: [Git - Install for Windows](https://git-scm.com/download/win).
- **الحساب:** نقوم بإنشاء حساب رسمي على [GitHub](https://github.com).

---

## 2. مرحلة الإعداد العام (مرة واحدة فقط)

لكي تصبح جميع المشاريع التي سنرفعها تتبع للحساب الخاص بك، افتح تطبيق **Git Bash** ونفذ الأوامر التالية:

```bash
git config --global user.name "(GitHub account name)"
git config --global user.email "(GitHub email)"

```

للتأكد من نجاح الإعدادات:

```Bash
git config --list

```

سيعرض لك الجهاز الاسم والايميل اللذين قمت بإدخالهما.

## 3. مرحلة تجهيز المستودع على GitHub

ادخل إلى حسابك في غيت هاب.

أنشئ مستودعاً جديداً بالضغط على New أو Create Repository.

ضع اسماً للمستودع وشرحاً بسيطاً، ثم اختر Public (عام) أو Private (خاص).

README: فَعّل خيار Add README لكتابة شرح مطول يظهر في واجهة المشروع.

تحديد الرخصة (License):

MIT License: للسماح للجميع بالاستخدام بشرط ذكر المصدر.

Apache License: إذا كنت تهتم بأمور براءات الاختراع.

GNU GPL: للحفاظ على البرمجيات مفتوحة المصدر للأبد.

CC0: للتنازل عن جميع الحقوق وجعلها ملكية عامة.

تنبيه: إذا لم تحدد رخصة، تبقى حقوق النشر افتراضية ولا يحق للغير استخدام الكود تقنياً.

## 4. مرحلة رفع المشروع

اذهب لمجلد المشروع في جهازك، ومن الخيارات الإضافية اختر Git Bash Here، ثم استخدم الأوامر بالترتيب:

```Bash
# تحويل المجلد لمشروع غيت
git init

# إضافة جميع الملفات للتجهيز
git add .

# تثبيت الملفات في نسخة محلية مع وصف
git commit -m "A description"

# تحديد الفرع الرئيسي
git branch -M main

# ربط المجلد المحلي بالمستودع أونلاين
git remote add origin (GitHub repository link)

# الرفع النهائي للموقع
git push -u origin main

```

## 5. إعداد مفتاح الوصول (Token)

سيطلب منك الموقع التوكين عند الرفع، اتبع الخطوات التالية:

اذهب إلى Settings من صورتك الشخصية.

اختر Developer settings ثم Personal access tokens ثم Tokens (classic).

اضغط Generate new token (classic).

اكتب أي اسم في الملاحظة، حدد المدة، وضَع علامة صح على خيار repo.

اضغط إنشاء وانسخ التوكين واحفظه فوراً في ملف خارجي (لن تراه ثانية).

## 6. ملاحظات هامة 💡

النسخ واللصق: استخدم أوامر الماوس (Right Click) داخل الـ Terminal.

رابط المستودع: هو رابط صفحة المستودع التي تظهر في المتصفح.

الرفع الإجباري: إذا رفض الموقع الرفع بسبب اختلاف الملفات، استخدم:
git push -u origin main --force

أوامر التحديث (للمرات القادمة):

```Bash
git add .
git commit -m "New Description"
git push origin main
```
